Server Requirements‚Äã

Shopify Laravel framework has a few system requirements. You will need to make sure your server meets the following requirements:

  1. PHP 7.4 (If need to upgrade or downgrade follow this link https://webdock.io/en/docs/perfect-server-stacks/upgrading-or-downgrading-php-versions)
  2. BCMath PHP Extension (sudo apt install php7.4-bcmath)
  3. Ctype PHP Extension (Installed by default as of PHP 5.3.0)
  4. Fileinfo PHP extension (Installed by default as of PHP 5.3.0)
  5. JSON PHP Extension (sudo apt install php7.4-json)
  6. Mbstring PHP Extension (sudo apt install php7.4-mbstring)
  7. OpenSSL PHP Extension (sudo apt install openssl)
  8. PDO PHP Extension (sudo apt-get install php7.4-mysql && sudo phpenmod pdo_mysql)
  9. Tokenizer PHP Extension (sudo apt install php7.4-tokenizer)
  10. XML PHP Extension (sudo apt install php7.4-xml)
  11. CURL PHP Extension (sudo apt install php7.4-curl)
  12. ZIP PHP Extension (sudo apt install php7.4-zip)
  13. GD PHP Extension (sudo apt install php7.4-gd)
  14. Enable SSL (sudo a2enmod ssl)
  15. Use letsencrypt for generating certificates (https://letsencrypt.org/docs/certificates-for-localhost/)
  16. memchache (https://devdocs.magento.com/guides/v2.4/config-guide/memcache/memcache_ubuntu.html)
  17. Phantomjs (https://www.vultr.com/docs/how-to-install-phantomjs-on-ubuntu-16-04)
  18. Enable php mod_rewrite (https://www.digitalocean.com/community/tutorials/how-to-rewrite-urls-with-mod_rewrite-for-apache-on-ubuntu-18-04)
  19. Make sure mod headers are installed
  20. sudo apt-get install php7.4-mailparse

Restart the apache server after installing the above mentioned extensions. For restarting use the below command.

  1. sudo service apache2 restart

Import test data‚Äã

First create 3 databases named as below:

  1. shopperapproved
  2. shared

Then import the .sql files provided in the package, using command line or using any client tool, for command line import follow the instructions given on the link https://stackoverflow.com/questions/17666249/how-do-i-import-an-sql-file-using-the-command-line-in-mysql

Then create mysql user using the command line and commands below:

  1. create user 'shopperapproved'@'%' identified by 'your_complex_password';
  2. grant ALL PRIVILEGES on . to 'shopperapproved'@'%';

Installing Laravel‚Äã

Before you start laravel installation, you need to configure your .env file and /config/shopperapproved.php, paste shopperapproved.php file one folder behind your root directory inside config directory and provide your correct paths and mysql credentials, for help I will send example.env file in the package that will be the same file as you will use except few changes of path where I have written ‚ÄúThis need to change‚Äù, then go to the laravel-develop folder, and install laravel modules/packages using the below command

  1. composer update or composer install
  2. Composer version should be 2.x

If composer update does not work then remove the vendor folder and run composer install.

After the successful installation, please run the following command from the laravel-develop

  • cp vendor/autoload.php ../vendor/autoload.php
  • This command will create a copy of autoload.php into the previous directory of vendor
  • Now open the autoload.php file from the target directory
  • And replace requireonce _DIR _. '/composer/autoload_real.php'; with require_once dirname(**_DIR**) . '/laravel-develop/vendor/composer/autoload_real.php';
  • Double check the path

When all the modules/packages are installed then set the permissions and ownership for the storage folder and laravel main folder, using the following commands

  1. cd ..
  2. sudo chmod -R 755 laravel-develop
  3. chmod -R o+w laravel-develop/storage

Run DB migrations and seeders to update the schema

Installing NPM & Node‚Äã

After the laravel installation, we have to install npm and node version 10.19.0, then execute below set of commands under [ROOT]/laravel-develop directory

  1. npm install
  2. npm run prod

If you are on the latest version of node then install Node Version Manager (nvm). After installing it, run the command nvm use 10.19.0 and nvm alias default 10.19.0

After the above commands now you are good to proceed to the ‚Äúbrowse website‚Äù section.
